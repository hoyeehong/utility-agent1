🎯 DEPLOYMENT WORKFLOW (5 Steps)
════════════════════════════════════════════════════════════════════════════

STEP 1: Create GitHub Repository (5 min)
────────────────────────────────────────────────────────────────────────────
Go to: https://github.com/new
Repository name: utility-bill-calculator
Create the repository

STEP 2: Push Code to GitHub (2 min)
────────────────────────────────────────────────────────────────────────────
cd /Users/hoyeehong/Desktop/demo
git remote add origin https://github.com/YOUR_USERNAME/utility-bill-calculator.git
git branch -M main
git push -u origin main

STEP 3: Create Railway Project (3 min)
────────────────────────────────────────────────────────────────────────────
Go to: https://railway.app/dashboard
Click: "New Project"
Select: "Deploy from GitHub repo"
Choose: utility-bill-calculator repository
Railway auto-detects Python and starts deployment

STEP 4: Configure Environment Variables (5 min)
────────────────────────────────────────────────────────────────────────────
In Railway Dashboard → Your Project → Variables tab, add:

SUPABASE_URL = https://your-project.supabase.co
SUPABASE_KEY = your-anon-key
ANTHROPIC_API_KEY = your-anthropic-api-key
TELEGRAM_BOT_TOKEN = your-telegram-bot-token
TELEGRAM_ADMIN_ID = your-telegram-user-id (optional)

Then click "Deploy" to redeploy with environment variables

STEP 5: Run Database Migration (2 min)
────────────────────────────────────────────────────────────────────────────
Go to: Supabase Dashboard → SQL Editor
Create: New Query
Copy: Contents of supabase/schema.sql
Execute: The SQL

DONE! Total time: 20-30 minutes ✅

🎯 WHAT HAPPENS AFTER DEPLOYMENT
════════════════════════════════════════════════════════════════════════════

1. User sends /start to Telegram bot
   ↓
2. Bot prompts for meter readings
   ↓
3. User sends electricity + water meter images
   ↓
4. FastAPI extracts readings via image OCR
   ↓
5. Orchestrator validates data
   ↓
6. TariffRateManager fetches current rates from Supabase
   ↓
7. UtilityCalculator computes bill (electricity + water breakdown)
   ↓
8. Formatter prepares Telegram message
   ↓
9. User receives bill with:
   • Electricity usage and charge
   • Water breakdown (usage + waterborne + conservation)
   • GST and surcharge
   • Total amount due
